# GitHub Integration

Connect your Epic Stack workspace to GitHub to manage repositories and issues seamlessly.

## Overview

The GitHub integration allows you to:

- Link notes to GitHub issues
- Create issues from notes
- Sync issue status and updates
- Manage repositories and projects
- Track work across platforms

## Setup Instructions

### 1. Create a GitHub OAuth Application

1. Go to [GitHub Developer Settings](https://github.com/settings/applications/new)
2. Click **New OAuth App**
3. Fill in the application details:
   - **Application name**: Epic Stack
   - **Homepage URL**: Your application URL
   - **Authorization callback URL**: `https://your-domain.com/api/integrations/oauth/callback`
4. Click **Register application**
5. Copy your **Client ID** and generate a **Client Secret**

### 2. Set Environment Variables

Add these to your `.env` file:

```bash
GITHUB_INTEGRATION_CLIENT_ID=your_github_client_id
GITHUB_INTEGRATION_CLIENT_SECRET=your_github_client_secret
INTEGRATION_ENCRYPTION_KEY=64_character_hex_string
```

### 3. Connect Your GitHub Account

1. Go to your organization settings
2. Navigate to the **Integrations** tab
3. Find **GitHub** and click **Connect**
4. You'll be redirected to GitHub's authorization page
5. Log in with your GitHub account
6. Click **Authorize** to grant permissions

### 4. Select Repositories

After connecting, select which GitHub repositories you want to work with:

- Choose default repositories for issue creation
- Configure default labels and assignees
- Set up repository-specific workflows

## Using the Integration

### Linking Notes to Issues

1. Open a note in Epic Stack
2. Click the **Link to GitHub** option
3. Search for or select a GitHub issue
4. The note will be linked to the issue

### Creating Issues from Notes

1. Create a note in Epic Stack
2. Click **Create GitHub Issue**
3. Select the repository
4. Configure issue details (title, description, assignee, labels, milestone, etc.)
5. Click **Create**
6. The issue will be created in GitHub and linked to your note

### Syncing Issue Status

Connected issues automatically sync:

- Status changes in GitHub appear in Epic Stack
- Comments and updates are reflected
- Assignee changes are synchronized

## Advanced Features

### Custom Fields

Map Epic Stack fields to GitHub custom fields:

- Configure field mappings in integration settings
- Sync custom field values automatically
- Support for various field types

### Workflow Automation

Set up automated workflows:

- Auto-create issues when notes reach certain status
- Update issue status based on note changes
- Trigger notifications on issue updates

## Troubleshooting

### Connection Issues

**Problem**: Authorization fails or shows permission errors

**Solution**:
- Ensure you have admin access to the GitHub organization
- Check that your GitHub account has API access enabled
- Verify your GitHub account is active
- Try disconnecting and reconnecting

### Issues Not Appearing

**Problem**: Expected GitHub issues don't show up in Epic Stack

**Solution**:
- Verify you have permission to view the issues in GitHub
- Check that the repository is selected in integration settings
- Refresh the integration connection
- Ensure the issue type is supported

### Sync Failures

**Problem**: Changes aren't syncing between platforms

**Solution**:
- Check your internet connection
- Verify the integration is still connected
- Review GitHub API rate limits
- Reconnect the integration

## Security & Privacy

- All communication with GitHub is encrypted
- Your GitHub tokens are securely stored and encrypted
- Epic Stack only accesses the permissions you explicitly grant
- You can revoke access at any time from GitHub's app management

## Disconnecting

To disconnect GitHub:

1. Go to organization settings
2. Navigate to **Integrations**
3. Find **GitHub** and click **Disconnect**
4. Confirm the disconnection

You can also revoke access from your GitHub account settings.
