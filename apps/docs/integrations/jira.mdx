# Jira Integration

Connect your Epic Stack workspace to Jira to link notes with issues, projects, and workflows.

## Overview

The Jira integration allows you to:

- Link notes to Jira issues
- Create issues from notes
- Sync issue status and updates
- Manage projects across both platforms
- Track work in real-time

## Setup Instructions

### 1. Create a Jira OAuth Application

1. Go to [Atlassian Developer Console](https://developer.atlassian.com/console/myapps/)
2. Click **Create** and select **OAuth 2.0 integration**
3. Fill in the application details:
   - **App name**: Epic Stack
   - **App URL**: Your application URL
4. Under **Authorization**, set your redirect URI: `https://your-domain.com/api/integrations/oauth/callback`
5. Under **Permissions**, request these scopes:
   - `read:jira-work` - Read access to Jira work items
   - `write:jira-work` - Write access to create and update issues
   - `manage:jira-project` - Project management permissions
6. Copy your **Client ID** and **Client Secret**

### 2. Set Environment Variables

Add these to your `.env` file:

```bash
JIRA_CLIENT_ID=your_jira_client_id
JIRA_CLIENT_SECRET=your_jira_client_secret
INTEGRATION_ENCRYPTION_KEY=64_character_hex_string
```

### 3. Connect Your Jira Account

1. Go to your organization settings
2. Navigate to the **Integrations** tab
3. Find **Jira** and click **Connect**
4. You'll be redirected to Atlassian's authorization page
5. Log in with your Jira account
6. Click **Authorize** to grant permissions

### 4. Select Projects

After connecting, select which Jira projects you want to work with:

- Choose default projects for issue creation
- Configure default issue types
- Set up project-specific workflows

## Using the Integration

### Linking Notes to Issues

1. Open a note in Epic Stack
2. Click the **Link to Jira** option
3. Search for or select a Jira issue
4. The note will be linked to the issue

### Creating Issues from Notes

1. Create a note in Epic Stack
2. Click **Create Jira Issue**
3. Select the project and issue type
4. Configure issue details (summary, description, assignee, etc.)
5. Click **Create**
6. The issue will be created in Jira and linked to your note

### Syncing Issue Status

Connected issues automatically sync:

- Status changes in Jira appear in Epic Stack
- Comments and updates are reflected
- Assignee changes are synchronized

## Advanced Features

### Custom Fields

Map Epic Stack fields to Jira custom fields:

- Configure field mappings in integration settings
- Sync custom field values automatically
- Support for various field types

### Workflow Automation

Set up automated workflows:

- Auto-create issues when notes reach certain status
- Update issue status based on note changes
- Trigger notifications on issue updates

## Troubleshooting

### Connection Issues

**Problem**: Authorization fails or shows permission errors

**Solution**:
- Ensure you have admin access to the Jira instance
- Check that your Jira account has API access enabled
- Verify your Jira instance URL is correct
- Try disconnecting and reconnecting

### Issues Not Appearing

**Problem**: Expected Jira issues don't show up in Epic Stack

**Solution**:
- Verify you have permission to view the issues in Jira
- Check that the project is selected in integration settings
- Refresh the integration connection
- Ensure the issue type is supported

### Sync Failures

**Problem**: Changes aren't syncing between platforms

**Solution**:
- Check your internet connection
- Verify the integration is still connected
- Review Jira API rate limits
- Reconnect the integration

## Security & Privacy

- All communication with Jira is encrypted
- Your Jira tokens are securely stored and encrypted
- Epic Stack only accesses the permissions you explicitly grant
- You can revoke access at any time from Jira's app management

## Disconnecting

To disconnect Jira:

1. Go to organization settings
2. Navigate to **Integrations**
3. Find **Jira** and click **Disconnect**
4. Confirm the disconnection

You can also revoke access from your Jira account settings.
